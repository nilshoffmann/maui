Installation of Maui

Download one of the following installers suitable for your operating system and run it to install Maui on your system.

Novice users should prefer one of the stable release versions below. Expert users and developers may also check the latest stable developer build here.

The source code of Maui is available at sourceforge.net.

Release History

Maui 1.3.0

Released on March xx, 2014

Requirements
  • Maui 1.3.0 requires a Java Runtime Environment (JRE) 7 in order to run. We recommend to install a 64 bit version of Java where possible to allow more memory to be allocated.
  • The statistical backend of Maui is based on R, at least version 3.0, and accessed using Rserve. Please consult the respective sites on notes how to install R for your operating system.
    Rserve may be installed from within a running R session via install.packages("Rserve").
New Features
  • Added action for peak group database identification.
  • Added combo boxes to select, which principle components are displayed.
  • Added FoldChangeViewer module to display volcano plots on pairs of treatment groups, based on anova p-values.
  • Added Action to create SVG application screenshots.
  • Added possibility to hide sample nodes in peak group and peak container views.
  • Added exception handling for unexpected exceptions, prematurely closing the database.
  • Added a default ehcache configuration.
  • Added additional actions for easy creation of user peak databases.
  • Added additional action for identification of selected peaks.
  • Added action to add a newly imported database to an existing database container.
  • Added svg export of charts.
  • Added xcms matched filter peak finding template.
  • Added project-specific peak normalization remembering the last selected group.
  • Added search ability for peaks.
  • Added project-specific peak pvalue adjustment remembering the last selected method.
  • Added database resize actions to Options panel.
  • Added panning (CTRL+MOUSE) and mouse-wheel zooming ability to all charts.
  • Added checking for peaks with empty mass spectra which are now skipped during conversion.
  • Added a guard against accidental null setting of PeakAnnotationDescriptor displayName.
  • Added a new Action to export the mass spectrum to the clipboard in a format compatible with Golm Metabolite Database's web frontend.
  • Added a project submenu provider and switched implementation for getActions in ChromAUIProjectNode to the new system.
  • Added flashing of scans in Chromatogram Views on selection from navigator view.
  • Added category datasets to charting support, adapted other classes to changes.
  • Added a warning dialog if different Descriptors should be deleted.
  • Added new Methods to IMauiProject for descriptor and container retrieval by id.
  • Added chromatogram creation to DescriptorFactory.
  • Added msLevel attribute to Scans on creation.
  • Added support for different radius for neighbor search in horizontal and vertical directions in 2D chromatogram views.
  • Added some convenience methods to IMauiProject for simpler retrieval of the output and import directories.
  • Added installed maltcms pipelines as dynamic actions into project context menu.
  • Added additional field for maltcms runtime arguments.
  • Added better error messages and progress indication to PeakAlignmentImporter.
  • Added CheckDatabaseConsistency Action to check a closed database for errors.
  • Added ui elements to change block size in Chromatogram2DPanel.
Updates and Improvements
  • Updated cross and maltcms versions to 1.3.1.
  • Updated cross and maltcms versions to 1.3.0.
  • Updated peak descriptor to include baseline information. Added rendering of peak shapes to peak overlay.
  • Updated navigator view for 1D chromatograms.
  • Updated JOGL version and Java3D due to Mesa-related rendering bug on Ubuntu with Intel graphics.
  • Updated pipeline wizard (not fully functional yet).
  • Updated license to include L-GPL and EPL license text.
  • Updated implementation of CachingChromatogram1D and 2D to be aware of ms-n information. Adapted charting datasets and dataset providers accordingly.
  • Updated DescriptorNode generation to also include contents of objects implementing Lookup.Provider.
  • This allows selection of PeakGroups and corresponding highlighting of associated PeakAnnotationDescriptors, i.e. in the Chromatogram1D View.
  • Updated PCA calculation to better map input data to processed pca data. Added co-plotting of projected variables (peak groups) in PCAViewer and interactive selection, visualized in PeakGroup TopComponent.
  • Improved speed of peak group import and retrieval of retention time related data.
  • Updated Chromatogram1DViewTopComponent to allow for visualization of visible/hidden chart series.
  • Updated the navigator display of peak overlays, now grouped by their parent chromatogram descriptor. All overlays for one parent can now be set visible/invisible all at once or individually.
  • Improved display of FileFragments and their variables to avoid loading of array data.
  • Updated wizard to allow creation of SampleGroups. Updated node factories to display sample groups if they are present.
Bug Fixes
  • Fixed chromatogram node menu item problem for multiple selections.
  • Fixed a startup problem on Linux for R>=3.0.
  • Fixed Navigator PanelUI to keep contents, expansion state and as much as possible, the selection.
  • Fixed import problems with latest GMD database format in MSP.
  • Fixed MassSpectrumPanel viewer to also process selection events from FoldChangeViewer.
  • Fixed copying of downloaded maltcms distribution to avoid an exception under windows.
  • Fixed the enablement logic into the finally block to avoid disabling the options dialog on failures/exceptions.
  • Fixed storage of normalization and pvalue adjustment dialogs to only use project name. Using the project path leads to problems on windows platforms.
  • Fixed search output to include project name and chromatogram name for each peak.
  • Fixed PeakGroupContainer to set itself as PeakGroupContainer on added IPeakGroupDescriptors.
  • Fixed a bug with caching of scans in CachingChromatogram1D and -2D.
  • Fixed a bug when opening 2D chromatograms in 1D view.
  • Fixed a minor issue with viewport synchronization.
  • Fixed index out of bounds problem in CachingChromatogram1D. Modified Peak1DOverlay so that it listens to global selection of PeakAnnotationDescriptors e.g. from the project explorer, and highlights the appropriate peaks.
  • Fixed issue [#13]. EIC Coplot is still not fully functional, but all other plots now work as expected. Added peak indicator lines to overlay. Can be customized via navigator node properties.
  • Fixed pipeline to scene parsing of PipelineEditor. Related to issue [#8].
  • Fixed problems with project wizard losing sample and treatment groups [#10]. Fixed layout differences.
  • Fixed wizard group definition issue [#7].
  • Fixed selection issues in PeakTableViewerTopComponent. Modified view to include source chromatograms of peak annotation descriptors.
  • Fixed getChromatogram() method to internally use URIs to align it further with IFileFragment.
  • Fixed a bug in retention time to scan index mapping [#3].
  • Fixed issues [#1]: Added better handling of missing sample group associations in the New Project Wizard.
  • Fixed retrieval of latest maltcms build.
  • Fixed a small issue in SelectionOverlay with antialiasing.
  • Fixed some layer registration problems. Improved behaviour of PeakGroupAnovaRunnable and PeakGroupPcaRunnable by adding progress information and updating the project node hierarchy.
  • Fixed: Input files are now passed as URIs to Maltcms.
Removed Functionality
  • Deactived meltdb module actions due to server-side encoding problems.

New Features
  • Peak overlays are now better visible for 1D chromatogram views
  • Peak overlays are colored according to peak descriptor container color, or by treatment group color
  • Added a diff button to mass spectral viewer to show a difference spectrum of two mass spectra
  • Peak groups now show a percentage of covered chromatograms
Bug Fixes
  • Fixed Rserve connection testing on Mac OS X related to the cairo library
  • Fixed invalid cache paths for Ehcache on Windows
  • Fixed EIC display options for 1D chromatogram views
  • Fixed label display of masses with highest intensities in mass spectral view
  • Changed update center locations, SF FRS location is incompatible with NetBeans updater
Known Issues
  • Retention time to peak index mapping for ChromaTOF data is disfunctional when importing peaks additionally to raw data.
